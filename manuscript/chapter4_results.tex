\paragraph{Why slope~$\approx1$ is expected.}
The sensitivity $g_i$ is the $L^2$ Riesz representative
with respect to the filtered-projected field $\bar\rho$.
The Taylor perturbation is applied to the raw variable $\rho$,
which passes through the Helmholtz filter and Heaviside projection
before entering $J$.
This filter--projection chain introduces a first-order relationship
between $\rho$-perturbations and $J$-changes,
so slope~$\approx1$ is the correct result for this formulation.
Slope~$\approx2$ requires differentiating through the full
filter--projection chain, i.e.\ a discrete adjoint~\cite{farrell2013};
this is a planned extension.
The OC and MMA optimisers depend only on gradient sign,
which the direction test confirms.

\subsection{Mesh sensitivity and user guidance}
\label{sec:mesh_conv}
As expected for non-convex density-based topology
optimisation~\cite{lazarov2016}, different mesh resolutions
converge to different local optima with different RMSE values.
This is not a software defect; it is a known property of the
non-convex optimisation landscape.
The $80\times20$ mesh is the validated primary configuration
for the benchmark problem.
Full mesh sensitivity data (including $160\times40$ results)
are in Supplementary~S1.

\paragraph{User guidance.}
Users should treat RMSE values as problem-specific
and mesh-specific; cross-mesh comparison of RMSE is not meaningful
for non-convex topology optimisation.
For production use, we recommend running multiple mesh resolutions
and selecting the design with the lowest RMSE,
or using a coarse-to-fine continuation strategy.
The framework supports arbitrary mesh sizes
via the \texttt{nx}, \texttt{ny} arguments.

\subsection{SUPG stabilisation verification}
\label{sec:supg_val}
SUPG stabilisation~\cite{donea2003,brooks1982} is applied to both
forward and adjoint transport equations.
The same element-wise parameter $\tau$~\cite{donea2003} is used
for both equations, following standard practice for
convection-dominated adjoint
problems~\cite{collis2002,nadarajah2000}.
The mean cell P\'{e}clet number on the $20\times5$ mesh
is $\Pe_h\approx1.2\times10^5$, confirming that stabilisation
is essential.

\paragraph{Forward SUPG.}
Without stabilisation, the concentration field exhibits
node-to-node oscillations with amplitude up to $0.4$.
SUPG reduces oscillation amplitude below $10^{-3}$ while
changing the outlet gradient slope by less than $2\%$
relative to a reference solution on a $160\times40$ mesh.
Full upwinding over-diffuses the profile (slope reduction $\approx30\%$)
and is not used.

\paragraph{Adjoint SUPG.}
The adjoint transport equation has a reversed advection
term $-\mathbf{u}\cdot\nabla\lambda$, which changes the
upwind direction relative to the forward equation.
A numerical test on the $20\times5$ mesh compares the
adjoint sensitivity field with and without SUPG:
the relative difference in $\|\lambda\|$ is $99.4\%$
and the Pearson correlation between the two fields is $0.75$,
confirming that adjoint SUPG is essential at
$\Pe_h\approx1.2\times10^5$, not merely a minor correction.
The sensitivity direction test (Pearson~$\geq0.80$;
Table~\ref{tab:adjoint_verif}) is reported with SUPG
applied to both equations.

\subsection{Linear gradient target benchmark}
The linear target $c_{\mathrm{target}}(y)=y/L_y$ was selected
because it provides an analytically interpretable outlet profile
that enables rigorous verification of adjoint sensitivities,
optimisation convergence, and end-to-end reproducibility.
The framework is agnostic to the target profile:
arbitrary outlet objectives are supported by changing
a single parameter (\texttt{target\_expr}),
as demonstrated by the seven example scripts in \texttt{examples/}.

\paragraph{Volume constraint and baseline.}
The optimisation is subject to the volume constraint
$V = \int\bar{\rho}\,\mathrm{d}\Omega / |\Omega| = V^* = 0.5$.
The unconstrained void channel ($\rho\equiv1$, $V=1.0 > V^*$)
achieves RMSE~$\approx0.007$ via passive transverse diffusion,
but violates this constraint and is therefore not a valid
comparator for the constrained optimisation problem.
The correct constrained baseline is uniform $\rho=V^*=0.5$
(RMSE~$=0.450$); relative to this baseline,
the optimiser achieves an $\rmseImprovement\%$ reduction
(RMSE~$0.450\to\rmseTopOpt$).
The void channel is listed in Table~\ref{tab:context}
for completeness with a constraint-violation note.
After $\nIterPrimary$ iterations on the $80\times20$ mesh,
the implementation achieves RMSE~$=\rmseTopOpt$
($\rmseTopOptPP$\,pp; Eq.~\eqref{eq:rmse}),
hydraulic resistance $\hydResTopOpt$\,Pa$\cdot$s/m$^2$,
and gray fraction $\grayFrac$.
A 600-iteration run yields RMSE~$=\rmseShortRun$ in under 30 minutes.

\subsection{Convergence and reproducibility}
\label{sec:convergence}

The objective history (Figure~\ref{fig:convergence}) and optimised
topology (Figure~\ref{fig:topology}) are consistent across all runs
from different random seeds, indicating that the default configuration
reliably finds a qualitatively similar local minimum.

\paragraph{Mesh sensitivity of the optimised design.}
The optimised topology varies across discretisations,
as expected for non-convex topology optimisation:
the $160\times40$ mesh finds a qualitatively different local minimum
with higher RMSE and higher gray fraction ($0.557$ vs $0.074$),
even with a resolution-scaled iteration budget
(Section~\ref{sec:mesh_conv}).
The macroscopic branching pattern is qualitatively similar
across meshes, which is the expected behaviour
for density-based topology optimisation~\cite{lazarov2016}.
All quantitative results in this paper are reported for the
$80\times20$ mesh only and should be interpreted as
results for this resolution.
