\subsection*{S0: Getting Started Guide}
\label{sec:s0}
This section provides step-by-step instructions for reproducing
all results in this paper.

\paragraph{Step 1: Prerequisites.}
Install Docker (\url{https://docs.docker.com/get-docker/}).
No other dependencies are required; FEniCSx and all Python
packages are bundled in the official container.

\paragraph{Step 2: Clone and run.}
\begin{lstlisting}[language=bash,breaklines=true]
git clone https://github.com/NisongMonyimba/micrograd
cd brinkgrad
docker run --rm -v ${PWD}:/root/brinkgrad -e OMP_NUM_THREADS=1 -e MPLBACKEND=Agg dolfinx/dolfinx:v0.7.3 bash -c "cd /root/brinkgrad && bash run_all.sh"
\end{lstlisting}

\paragraph{Step 3: Quick start (single result).}
To reproduce only the primary linear gradient result (Figure~2,
approximately 1 hour):
\begin{lstlisting}[language=bash,breaklines=true]
docker run --rm -v ${PWD}:/root/brinkgrad dolfinx/dolfinx:v0.7.3 bash -c "cd /root/brinkgrad && python scripts/optimize.py --target linear --iter 1400"
\end{lstlisting}

\paragraph{Step 4: Modify parameters.}
Key parameters are set via command-line arguments:
\begin{verbatim}
# Change volume fraction
python scripts/optimize.py --target linear --V_star 0.3

# Change target profile
python scripts/optimize.py --target sigmoid --iter 600

# Change mesh resolution
python scripts/optimize.py --target linear --nx 160 --ny 40
\end{verbatim}

\paragraph{Step 5: Jupyter notebook.}
An interactive walkthrough is available in
\path{notebooks/quickstart.ipynb}:
\begin{lstlisting}[language=bash,breaklines=true]
docker run --rm -p 8888:8888 -v ${PWD}:/root/brinkgrad dolfinx/dolfinx:v0.7.3 bash -c "cd /root/brinkgrad && jupyter notebook --ip=0.0.0.0"
\end{lstlisting}

\paragraph{Expected outputs.}
After \texttt{run\_all.sh} completes, results are in:
\begin{itemize}\itemsep1pt
    \item \texttt{figures/}: all PDF and PNG figures
    \item \texttt{results/}: JSON files with RMSE, J, gray fraction
    \item \texttt{logs/}: convergence history per run
\end{itemize}

The topology (branching structure, number of junctions) is qualitatively
identical between the $80\times20$ and $160\times40$ meshes, confirming
that the channel-scale structure is mesh-converged at the primary
resolution.
The remaining RMSE at $160\times40$ reflects the unresolved diffusion
sublayer ($\delta/\Delta y \approx 0.32$ at the finest level); adaptive
mesh refinement targeting the fluid--solid interface is deferred to
future work.

For larger meshes, the direct MUMPS solver is replaced by a
block-triangular field-split preconditioner: algebraic multigrid (HYPRE)
for the velocity block and the Schur complement approximation for the
pressure~\cite{elman2014}.
Iteration counts remain below 50 for all mesh sizes tested
($20\times5$ to $160\times40$), confirming near-optimal scalability.

\subsection*{S10: Adjoint Gradient Verification Details}
\label{sec:s10}

Two verification tests are reported in Section~\ref{sec:taylor_test}.

\paragraph{FD Taylor remainder.}
The finite-difference directional derivative
$\hat{g}[\delta\rho] = (J(\rho+\varepsilon\delta\rho)-J(\rho))/\varepsilon$
at $\varepsilon=10^{-5}$ is used as the reference.
The Taylor remainder $R(\varepsilon) = |J(\rho+\varepsilon\delta\rho)-J(\rho)-\varepsilon\hat{g}[\delta\rho]|$ is computed for
$\varepsilon\in\{10^{-5},\ldots,10^{-2}\}$.
FD remainder slope: $1.01$ ($20\times5$), $1.00$ ($80\times20$).
The slope of 1 reflects that the Riesz representative
does not satisfy the quadratic Taylor remainder condition;
the Pearson direction correlation $\geq 0.80$ confirms
correct descent direction for OC/MMA.

\paragraph{Direction test.}
The mass-scaled adjoint $g/m_i$ is compared to the finite-difference
gradient $\hat{g}_i$ component-wise on 20 sampled DOFs.
Pearson correlations: $0.80$ ($20\times5$), $0.88$ ($80\times20$).
The Euclidean gradient $\hat{g}=M^{-1}g$ (consistent mass solve,
CG/Jacobi, $\mathrm{rtol}=10^{-10}$) is verified on the
$20\times5$ mesh: single-DOF relative error reduces from
$>100\%$ (Riesz $g$) to $\approx31\%$ ($\hat{g}$).
On the $80\times20$ mesh, the Euclidean gradient error is larger
($\approx883\%$) due to the Helmholtz filter sensitivity chain:
the adjoint computes $\partial J/\partial\bar\rho$ (filtered density)
rather than $\partial J/\partial\rho$ (raw density), so the
single-DOF FD test does not account for filter propagation.
This reflects the filter chain in filtered topology optimisation
sensitivity analysis~\cite{lazarov2016}.
OC and MMA use only gradient direction; convergence is
validated by the pure-MMA comparison (Table~\ref{tab:mma_comparison}).

\paragraph{Continuous vs.\ discrete adjoint.}
The assembled sensitivity is a continuous adjoint
$g_i=\int(\partial J/\partial\bar\rho)\psi_i\,dx$,
not the discrete gradient $\hat{g}_i=\partial J/\partial\bar\rho_i$.
The relative error $E_g = \|g/m - \hat{g}\|/\|\hat{g}\| = 18.9$
($20\times5$) reflects this mismatch. A discrete adjoint is deferred
to future work.

\subsection*{S5: Thresholding Robustness and Binary Gap Analysis}
\label{sec:s5}

The optimised design has a gray-zone fraction of $\grayFrac$ (measured at $\beta=64$, 1000-iteration continuation) at
$\beta=\betaMax$, reflecting the physical reality that the linear
gradient target is achieved by distributed diffusive mixing through
intermediate-density Brinkman elements.
Threshold-projection at $\bar\rho=0.5$ and re-solving the forward problem
gives the results in Table~\ref{tab:binary_val}.

\begin{table}[h]
\centering
\caption{Binary validation results on the $80\times20$ mesh.
         Both binary configurations give identical RMSE, confirming
         the gap is caused by loss of gray mixing pathways, not
         by the Brinkman penalisation in fluid regions.}
\label{tab:binary_val}
\begin{tabular}{@{}lccc@{}}
\toprule
\textbf{Design} & \textbf{RMSE} & \textbf{$\Delta$RMSE vs gray} & \textbf{Method} \\
\midrule
Gray Brinkman   & $\rmseTopOpt$         & ---              & Smooth $\alpha(\bar\rho)$ \\
Binary Brinkman & $\rmseBinaryBrinkman$ & $+\binaryGapPct\%$ & $\bar\rho\in\{0,1\}$, smooth $\alpha$ \\
\bottomrule
\end{tabular}
\end{table}

\noindent
The $+\binaryGapPct\%$ RMSE change is mechanistically expected:
the linear gradient target $c^\star(y)=y/L_y$ is optimally achieved
by diffusive transport through intermediate-density elements, which
have no binary physical analogue.
The gap is not caused by the Brinkman penalisation in fluid
regions but by the loss of the gray diffusive mixing pathway.
The analytical bound $t_{\mathrm{diff}}/t_{\mathrm{flow}} \approx 3000$
(Section~\ref{para:leakage}) confirms that solid leakage through
$D_{\min}$ is negligible; the binary gap is a property of the
topology, not a numerical artefact.
Body-fitted Navier--Stokes validation on a purpose-designed binary
multi-channel manifold is deferred to future work.
